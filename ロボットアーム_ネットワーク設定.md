# ロボットアーム × インターネット 同時接続メモ（Macmini）

最終更新: 2026-06-05

## 結論（最終構成）

| 用途 | インターフェース | 設定 |
|---|---|---|
| インターネット | **en1 (Wi-Fi)** | DHCP / gateway `192.168.1.1`（サブネット `192.168.1.0/24`） |
| ロボットアーム | **en0 (Ethernet)** | 手動 IP `192.168.2.100` / mask `255.255.255.0` / gateway 空 |
| アーム本体 | — | 固定 IP `192.168.2.156` / mask `255.255.255.0` / gateway `192.168.2.1` |

- **UFACTORY Studio:** http://192.168.2.156:18333
- サブネットを分離（`192.168.1.x` ⇄ `192.168.2.x`）したので衝突は起きない。**再起動後も維持される。**

## 何が問題だったか

Ethernet でアームに繋ぐとインターネットが切れていた。原因は2段階：

1. **デフォルト経路の奪い合い** … Ethernet 側にゲートウェイが設定されていて、インターネット宛もアーム側ネットワークに流れていた。
2. **サブネット衝突** … Wi-Fi もアームも同じ `192.168.1.0/24` にいたため、アーム宛 (`192.168.1.156`) が Wi-Fi(en1) に吸われて `No route to host` になっていた。

## やった対策

1. Wi-Fi をサービス順序で Ethernet より上に（インターネットを Wi-Fi 優先に）。
2. **アームを別サブネットへ移動**して衝突を根絶：
   - アーム本体の IP を `192.168.1.156` → `192.168.2.156` に変更（UFACTORY Studio のネットワーク設定）。
   - Mac の en0 を `192.168.2.100 / 255.255.255.0`・ゲートウェイ空 に変更。

> メモ: UFACTORY Studio で IP 変更時に「Address is not standardized」が出たら、
> ゲートウェイ欄を空や `0.0.0.0` にせず **同サブネットの有効値（`192.168.2.1`）** を入れる。
> 数字は必ず**半角**（日本語IMEの全角だと弾かれる）。

## 動作確認コマンド

```bash
route -n get default          # → interface: en1（ネットは Wi-Fi）
route -n get 192.168.2.156    # → interface: en0（アームは Ethernet）
ping -c 2 8.8.8.8             # インターネット疎通
ping -c 2 192.168.2.156       # アーム疎通
```

両方の ping が 0% packet loss なら正常。

## トラブル時の切り分け

- アームに繋がらない → `route -n get 192.168.2.156` が `en0` か確認。`en1` なら経路がおかしい。
- インターフェース対応の確認 → `networksetup -listallhardwareports`
  - 現状: **en0 = Ethernet（アーム）/ en1 = Wi-Fi（ネット）**
- 古い静的 ARP/ルートが悪さをした場合の掃除:
  ```bash
  sudo arp -d 192.168.2.156
  sudo route delete 192.168.2.156
  ```
- 一時的に切り替えて使いたいだけなら（設定変更なし）:
  ```bash
  networksetup -setairportpower en1 off   # アーム専念
  networksetup -setairportpower en1 on    # ネット復帰
  ```

## 注意

- 他の PC からアームへ繋ぐ場合、アドレスは **`192.168.2.156`**（旧 `192.168.1.156` ではない）。
- アーム側サブネットは `192.168.1.x`（Wi-Fi）と被らなければ何でも可（例: `192.168.100.x` 等）。
